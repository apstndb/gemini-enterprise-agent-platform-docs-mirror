The following tutorials step through the various stages of a Agent Platform Neural Architecture Search run. To help you integrate your docker with the Agent Platform Neural Architecture Search service, we encourage you to *run* these tutorials, not only read through them. This process will then help you to integrate your own docker with the Agent Platform Neural Architecture Search service.

These tutorials are lightweight and only require a few cloud CPUs to run, which your project should have by default without any additional quota needs.

1.  [Run baseline training on Google Cloud using your docker](https://github.com/google/vertex-ai-nas/blob/main/third_party/tutorial/vertex_tutorial1.md) .
2.  [Create search spaces](https://github.com/google/vertex-ai-nas/blob/main/third_party/tutorial/vertex_tutorial2.md) .
3.  [Run an architecture search on Google Cloud](https://github.com/google/vertex-ai-nas/blob/main/third_party/tutorial/vertex_tutorial3.md) .
4.  [Add latency constraint to search: FLOPS or device based](https://github.com/google/vertex-ai-nas/blob/main/third_party/tutorial/vertex_tutorial4.md) .

After running the tutorials, you should read the [best practices and suggested workflow](https://docs.cloud.google.com/gemini-enterprise-agent-platform/machine-learning/training/neural-architecture-search/suggested-workflow) before running the first Agent Platform Neural Architecture Search search.
